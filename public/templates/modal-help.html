RepoGrams Help

RepoGrams will help you understand a project's history. It uses a simple visual metaphor to represent a project's history across multiple metrics of interest. It supports metrics like:

  • Who made commits, how often, and how much code did their commits change
  • The number of programming languages used in a project over time
  • How often developers changed code across different modules (directories)
    • RepoGrams can help:
    1. Managers to analyze a team's performance and pin-point workflow issues
    2. Developers to understand the workflow and the history of an unfamiliar software project
    3. Researchers in the software engineering community to juxtapose software repositories of multiple projects to select an appropriate set of evaluation targets for a study

    How to Use RepoGrams

    Add a repository and select a metric

    To begin using RepoGrams, you can first enter a git URL of the repository that you want to look at in the space below:

    Next, you can click on the "Switch" button beside "Metric" to enable a pop-up menu with a list of all the available metrics.

    Each metric will calculate the result for each commit independently for the repositories that you have selected. The displayed results are color coded per commit block and each color corresponds to a value in the legend.
    Group the results

    There are two options to choose from when grouping your results:

  • "Group by metric" will group your results by the metrics you selected. For example when you have multiple repositories loaded, it will display the results of each repository under each different metric.
  • "Group by repository" will group the results by repository instead.

    Choose a block length

    There are 3 choices for scaling displayed block widths:
  • Fixed block width: will set each commit block width to be the same for each repository.
  • Linear block width: will scale the width of each commit block linearly with respect to the number of lines changed in the commit.
  • Logarithmic block width: will scale the width of each commit block logarithmically with respect to the number of lines changed in the commit.

    • There are also 2 choices for comparing results between repositories:
  • Globally Normalized: This option will initally set all commit block widths to be the same before the scaling so that results are comparable between repositories.
  • Project Normalized: This option will optimize the commit block widths for each individual repository so that each result fills the screen.
    •
    Supported interactions
  • Commit Pop-up: Once the diagram is displayed, you can click on an individual commit to see a pop-up box with the commit ID and it's message as well as the values of each selected metric for that commit.
  • Zoom: If you have difficulty seeing some commits because they are too narrow try using the zoom slider at the top right of the page to enlarge the commit blocks!
  • Saving Your Session: If you want to save your work and come back to it another time, you can save your session by clicking "Load/save state" and downloading your session.
